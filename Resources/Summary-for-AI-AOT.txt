# UVF.NET Project Summary

The library will be called "TitanVault" for our users, the software developers

## Project Overview

**UVF.NET** is a C# implementation of the **Universal Vault Format (UVF)** specification, providing client-side encryption and decryption for secure file storage. This is a .NET port of an original Java library (Cryptomator's cryptolib-java) that enables applications to encrypt files and filenames before storing them (e.g., in cloud storage) and decrypt them upon retrieval.

### Key Goals:
- ✅ .NET-native implementation of the UVF standard
- ✅ Enable C# applications to create, read, and manage UVF-compatible encrypted vaults
- ✅ Maintain cryptographic compatibility with the original Java library
- 🚧 **Prepare for native C-style exports** for cross-language compatibility

## Current Status: Password Security Refactoring (COMPLETED ✅)

### What We Just Accomplished:
We have **successfully completed a comprehensive password security refactoring** across the entire UVF.NET codebase, converting all password handling from insecure `string` parameters to secure `byte[]` UTF-8 encoded parameters.

### Why This Was Critical:
1. **Security**: Strings are immutable in .NET and remain in memory until garbage collection
2. **Native Export Compatibility**: C-style exports require `IntPtr` buffers, not .NET strings
3. **Cross-Language Compatibility**: UTF-8 byte arrays work universally across all programming languages
4. **Performance**: Direct byte array usage eliminates UTF-8 encoding/decoding overhead

## Key Technical Achievements

### 1. **Secure Password Handling (✅ Complete)**
- **Before**: `string password` parameters (insecure, GC-dependent cleanup)
- **After**: `byte[] passwordBytes` parameters with `CryptographicOperations.ZeroMemory()` cleanup
- **Impact**: Eliminates password security vulnerabilities across 25+ critical methods

### 2. **Multi-User Vault Architecture (✅ Complete)**
- **Location**: `UvfLib.Core/Jwe/MultiUserJweVaultManager.cs`
- **Features**:
  - JWE-based multi-user encryption
  - Admin and user role management
  - Configurable key derivation (PBKDF2, Scrypt)
  - User addition/removal without re-encryption

### 3. **Unified Vault Format Support (✅ Complete)**
- **UVF v3**: Native multi-user format
- **Cryptomator V8**: Compatibility layer for existing vaults
- **Seamless Integration**: Single API supports both formats

### 4. **Native Export Preparation (✅ Ready)**
- **API Design**: All methods now accept `byte[]` passwords compatible with `IntPtr` marshaling
- **Memory Safety**: Proper cleanup of sensitive data
- **Cross-Language Ready**: UTF-8 byte arrays work with C, C++, Go, Rust, etc.
